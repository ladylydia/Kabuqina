## 1. 壳前端：ESLint 与 `tsconfig` 纳入 `vite.config.ts`

**背景**：在 `web/` 引入 **ESLint 9** 扁平配置（`eslint.config.js`，`typescript-eslint` 等）后，对 `vite.config.ts` 做类型感知的 lint 时，需让 **TypeScript 项目服务** 能发现该文件。

**改动**：

- **`web/tsconfig.json`**：`include` 在原有 `"src"` 基础上增加 **`"vite.config.ts"`**。

**结果**：`cd web && npm run lint` 与 `npm run build` 均可通过（以本机当时环境为准）。

---

## 2. Tauri：`cmd_get_hermes_port` 与 Clippy

**背景**：在 `cargo clippy -- -D warnings` 下，原先对 `Option<u16>` 使用 `clone_on_copy` 的写法已调整；随后采用 `Ok(*state.hermes_port.lock().await)` 时，在 **async 块末尾** 触发 Rust **E0597**（`State` 与 `MutexGuard` 临时量析构顺序）。

**改动**（**`tauri/src/lib.rs`**）：

- 先 **`let port = *state.hermes_port.lock().await;`** 再 **`Ok(port)`**，使锁在单条语句内释放，再返回 `Result`。

**结果**：`cd tauri && cargo clippy -- -D warnings` 通过。
